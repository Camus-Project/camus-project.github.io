## TL;DR

Camus is an open initiative building a unified method, specification language, and tooling for software generated by AI but still understood and assumed by humans. The project develops its methods and tooling by applying them to itself — a self-hosted approach to method engineering.

**Camus Method** is active and defines how to conceptualize, document, and certify AI-generated code through Lexicon, Grammar, and Certification phases. **Camus Skill** is an embeddable skill for AI-assisted workflows that implements the Method; it is under active development.

Future components include **Camus SL** (an embeddable specification language), **kiss** (a companion CLI tool), and **Camus PL** (a programming language focused on readability and explicit semantics). Each will be developed when the Method and SL have reached sufficient maturity.

If you want to dive straight in, read the [Camus Method overview](/method) or try the [Camus Skill](https://github.com/Camus-Project/camus-skill).

**The Camus Project is in early development.**
Specifications and tooling are still evolving. Join the community to help shape the design.

## Why Camus?

Software created with or assisted by AI requires clearer ways to record intent and responsibility. Camus aims to:

- Make intention explicit in specifications and code
- Capture constraints and expected behavior
- Keep human responsibility and review central
- Produce auditable artifacts for certification and verification

## How it works (brief)

- **Method**: A three-phase framework — Lexicon (define terms), Grammar (write to rules), Certification (human signature).
- **SL**: An embeddable spec language to express intent in a form understandable by humans and AI (future).
- **kiss**: A companion tool for validation, generation, and certification (future).
- **PL**: A programming language focused on readability, explicit semantics, and long-term maintainability (future).
- Humans validate and take responsibility before signing.

## Roadmap (high level)

- Stabilize the Camus Method and its certification workflow
- Define Camus SL as an embeddable specification language
- Implement kiss as a Cargo subcommand (`cargo kiss init`)
- Explore Camus PL as a future programming language
- Develop validation and certification tooling
