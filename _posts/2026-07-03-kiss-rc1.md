**kiss.sh** — the companion tool for applying the Camus Method to Bash — has reached RC1 and now lives in its own repository.

## What is kiss.sh?

kiss.sh automates the Camus.sh workflow:

- **Compliance checking** — validates scripts against the Camus.sh specification
- **Cryptographic signing** — signs reviewed functions with Ed25519 keys
- **Signature verification** — verifies attestation integrity and key validity

It is the shell adaptation of the `kiss` tool and its first prototype.

## What changed

### New repository

kiss.sh has been promoted from a subdirectory of Camus.sh to a standalone GitHub repository: [github.com/Camus-Project/kiss.sh](https://github.com/Camus-Project/kiss.sh).

This separation reflects the tool's maturity and makes it easier to install, distribute, and contribute to independently.

### Camus.sh archived

The original Camus.sh repository has been archived. Its content — the specification, agent skill, and related documentation — now lives on this site under [adaptations/Camus.sh](/adaptations/Camus.sh/).

### Site restructured

The site now hosts documentation pages for the core components:

- [The Camus Method](/The-Camus-Method/) — the foundational methodology
- [Camus Specification Language](/Camus-SL/) — the embeddable spec language
- [Camus.sh](/adaptations/Camus.sh/) — the shell scripting adaptation

## What's next

kiss.sh RC1 is a release candidate. The tool is functional but may still have rough edges. Feedback, bug reports, and contributions are welcome via the [issue tracker](https://github.com/Camus-Project/kiss.sh/issues).

Future work includes:

- Porting kiss to Rust (`Camus.rs`) for broader platform support
- Expanding compliance checks
- Improving the signing workflow
